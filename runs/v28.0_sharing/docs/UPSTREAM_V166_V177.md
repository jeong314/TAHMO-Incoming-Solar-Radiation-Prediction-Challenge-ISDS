### v17.7 method

v17.7 is a station reliability filter on top of v17.6.

It starts from the v17.6 dual correction blend and measures station-level validation harm. Stations with positive weighted validation degradation are reverted to the v16.6 base prediction.

Selected v17.7 candidate from `metrics.json`:

## Practical GitHub packaging recommendation

For a compact, reproducible v28.0 GitHub package, use the current approach:

1. Include the v28.0 final-stage script and documentation in `v28.0_sharing`.
2. List `v166_*`, `v177_*`, and other CSV inputs in `data/ARTIFACT_MANIFEST.csv` with sizes and SHA256 hashes.
3. Include the v16.6 and v17.7 generator scripts under `upstream_scripts/`.
4. Keep large CSV artifacts outside normal Git history or distribute them through Git LFS / release assets.
5. Document the upstream dependency chain here, instead of forcing reviewers to run every historical experiment from raw data.

This makes the submitted v28.0 file exactly reproducible while still explaining where the intermediate prediction artifacts came from.
